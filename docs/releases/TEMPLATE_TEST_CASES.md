### UT-001: 专家注册功能

**模块**：`ExpertRegistry`  
**函数**：`register()`

| 步骤 | 输入 | 预期输出 | 实际输出 | 状态 |
|------|------|---------|---------|------|
| 1 | 有效专家对象 | `Ok(())` | | ⬜ |
| 2 | 重复专家 ID | `Err(ExpertExists)` | | ⬜ |
| 3 | 空专家对象 | `Err(InvalidExpert)` | | ⬜ |

**代码**：
```rust
#[test]
fn test_register_expert() {
    let mut registry = ExpertRegistry::new();
    let expert = create_test_expert("psychology");
    assert!(registry.register(expert).is_ok());
}

#[test]
fn test_register_duplicate_expert() {
    let mut registry = ExpertRegistry::new();
    let expert1 = create_test_expert("psychology");
    let expert2 = create_test_expert("psychology");
    registry.register(expert1).unwrap();
    assert!(registry.register(expert2).is_err());
}
```

## 缺陷记录

### BUG-001: 专家激活后 Persona 未更新

**严重程度**：🔴 高  
**状态**：已修复 / 待验证 / 已关闭

**描述**：
激活专家后，Persona 配置没有正确更新为专家的配置。

**复现步骤**：
1. 启动服务
2. 调用 `POST /experts/psychology/activate`
3. 调用 `GET /persona`
4. 返回的仍是默认 Persona

**预期行为**：
返回专家的 Persona 配置（暖心心理咨询师）

**实际行为**：
返回默认 Persona 配置（Subhuti-Default）

**修复方案**：
在 `activate()` 函数中添加 `update_persona()` 调用。

**验证结果**：
- [ ] 修复后测试通过
- [ ] 回归测试通过

---

### BUG-002: 并发激活导致数据竞争

**严重程度**：🟡 中  
**状态**：待修复

**描述**：
多个请求同时激活不同专家时，出现数据竞争警告。

**复现步骤**：
1. 使用 `wrk` 发送 100 并发激活请求
2. 观察日志中的 Thread Sanitizer 警告

**影响范围**：
- 高并发场景
- 多用户环境

**临时方案**：
使用 `Arc<RwLock>` 保护专家状态。
