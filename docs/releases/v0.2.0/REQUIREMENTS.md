## 版本概述

### 版本目标

增强日志查询 API，支持日志级别过滤、时间范围查询和分页功能，提升线上问题排查效率。

### 范围

**包含**：
- ✅ 日志级别过滤（INFO/WARN/ERROR/DEBUG）
- ✅ 时间范围查询（start_time ~ end_time）
- ✅ 分页支持（limit/offset）
- ✅ API 文档更新

**不包含**：
- ❌ 实时日志流（WebSocket）
- ❌ 日志统计分析面板

## 用户故事

### US-001: 作为运维人员，我希望能快速查看错误日志

**场景**：
1. 调用 `GET /logs?level=ERROR`
2. 返回最近 50 条 ERROR 级别日志
3. 包含时间戳、日志内容、Trace ID

**验收条件**：
- [ ] 支持 level 参数
- [ ] 默认返回最新日志
- [ ] 响应时间 < 200ms

---

### US-002: 作为开发者，我希望能按时间范围查询日志

**场景**：
1. 调用 `GET /logs?start=2026-06-28T10:00:00Z&end=2026-06-28T11:00:00Z`
2. 返回指定时间范围的日志
3. 支持分页

**验收条件**：
- [ ] 支持 ISO 8601 时间格式
- [ ] 时间范围验证
- [ ] 分页参数正确
