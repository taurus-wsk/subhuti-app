## 🚀 发布步骤

### 发布前（T-1 天）

1. [ ] 冻结代码分支
2. [ ] 执行 `make release-test`
3. [ ] 生成测试报告
4. [ ] 更新文档
5. [ ] 通知相关人员

### 发布日（T-0）

1. [ ] 打标签
   ```bash
   git tag -a v0.x.0 -m "Release v0.x.0"
   git push origin v0.x.0
   ```

2. [ ] 构建 Docker 镜像
   ```bash
   docker build -t subhuti:v0.x.0 .
   docker tag subhuti:v0.x.0 subhuti:latest
   ```

3. [ ] 部署到生产环境
   ```bash
   docker stop subhuti-app
   docker rm subhuti-app
   docker run -d --name subhuti-app -p 8080:8080 subhuti:v0.x.0
   ```

4. [ ] 验证部署
   ```bash
   curl http://localhost:8080/subhuti/api/v1/health
   ./version-test-v0.x.0.sh quick
   ```

### 发布后（T+1 天）

1. [ ] 监控服务稳定性
2. [ ] 收集用户反馈
3. [ ] 处理紧急问题
4. [ ] 编写发布总结

## 🔄 回滚方案

### 回滚触发条件

- 服务启动失败
- 核心功能不可用
- 严重性能退化
- 数据一致性问题

### 回滚步骤

```bash
# 1. 停止当前版本
docker stop subhuti-app
docker rm subhuti-app

# 2. 启动上一版本
docker run -d --name subhuti-app -p 8080:8080 subhuti:v0.x-1.0

# 3. 验证回滚
curl http://localhost:8080/subhuti/api/v1/health
```

### 回滚检查

- [ ] 服务恢复正常
- [ ] 数据完整性验证
- [ ] 用户通知
