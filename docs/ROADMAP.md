## 🎯 近期目标 (v1.1 - v1.5)

### P0: 提示词管理系统（最高优先级）

**目标**：统一管理所有提示词，解决散落在代码中的问题

- [ ] **提示词模板引擎** - 变量注入、条件渲染、模板继承
- [ ] **提示词版本管理** - 版本历史、回滚、A/B 测试
- [ ] **按模块组织** - Skill/Flow/Expert 各自的提示词命名空间
- [ ] **多语言支持** - 中英文提示词切换
- [ ] **动态组装** - 运行时根据上下文拼接系统提示词
- [ ] **提示词调试** - 实时预览最终组装结果

**设计思路**：
```
prompt/
  ├── system/         # 系统级提示词
  ├── skills/         # 各 Skill 的提示词
  ├── flows/          # 各 Flow 的提示词
  ├── experts/        # 各专家的人设提示词
  └── soul/           # 心灵层相关提示词
```

---

### P0.5: 缓存系统

**目标**：降低 LLM 调用成本，提升响应速度

- [ ] **LLM 响应缓存** - 精确匹配缓存 + 语义相似度缓存
- [ ] **Embedding 缓存** - 相同文本重复利用向量
- [ ] **工具结果缓存** - 相同参数的工具调用复用结果
- [ ] **多级缓存** - 内存缓存 → Redis → 持久化
- [ ] **缓存策略** - TTL、LRU、容量限制
- [ ] **缓存统计** - 命中率、节省成本统计

---

### P1: 示例专家插件

**目标**：开发 2-3 个高质量示例专家插件，展示框架能力

- [ ] **心理咨询专家插件** - 完整展示人格注入 + 情感记忆 + CBT 技术
- [ ] **编程助手插件** - 展示代码理解 + 文件操作 + 多轮对话
- [ ] **学习伙伴插件** - 展示记忆管理 + 进度追踪 + 个性化推荐
- [ ] **插件开发模板** - 一键创建新插件的脚手架

**预期效果**：
- 验证插件系统的完整性
- 为其他开发者提供参考范例
- 展示框架的差异化能力

---

### P1.2: 评估/评测模块

**目标**：自动化评测 Agent 表现，数据驱动优化

- [ ] **评测数据集管理** - 用例集、标注、分类标签
- [ ] **多维度评分** - 准确性、相关性、安全性、风格一致性
- [ ] **自动评测** - LLM-as-judge + 规则校验混合评分
- [ ] **回归测试** - 版本间对比，防止性能退化
- [ ] **评测报告** - 可视化报表、趋势分析
- [ ] **在线评测** - 生产环境灰度 A/B 测试

---

### P1.5: 数据库深度整合

**目标**：将心灵宫殿和记忆系统完全持久化到 PostgreSQL + pgvector

- [ ] **记忆持久化** - 所有记忆存储到 PostgreSQL
- [ ] **向量搜索** - 使用 bge-m3 生成向量，pgvector 进行语义搜索
- [ ] **人格持久化** - 性格数据、演化历史存数据库
- [ ] **多用户支持** - 按 user_id 隔离数据
- [ ] **迁移工具** - 内存数据迁移到数据库

**技术方案**：
```
内存模式（开发/测试）  ←→  数据库模式（生产）
     Memory                    DatabaseStore
       ↓                            ↓
  MemoryPalace                MemoryPalace
  （相同接口）                 （相同接口）
```

---

### P1.8: 限流与配额管理

**目标**：生产环境必备的流量控制

- [ ] **按用户限流** - 每用户每分钟/每小时请求数限制
- [ ] **按 API Key 配额** - 不同等级的 Token 配额
- [ ] **全局限流** - 系统整体 QPS 保护
- [ ] **IP 级限流** - 防止恶意刷接口
- [ ] **费用统计** - 按用户/按项目统计 Token 消耗
- [ ] **超限处理** - 降级策略、排队机制、错误提示

**目标**：丰富默认技能库，开箱即用

- [ ] **文件操作增强** - 写入文件、删除文件、文件元信息
- [ ] **网络搜索** - 对接搜索引擎（Serper、Bing 等）
- [ ] **代码执行** - Docker 沙箱执行 Python/JS 代码
- [ ] **定时提醒** - 持久化提醒、到期推送

### P2.2: 工具沙箱系统

**目标**：安全隔离工具执行环境

- [ ] **代码执行沙箱** - Docker 容器隔离执行代码
- [ ] **文件系统隔离** - 工具只能访问指定目录
- [ ] **权限控制** - 细粒度的工具权限管理
- [ ] **资源限制** - CPU、内存、执行时间限制
- [ ] **网络隔离** - 控制工具的网络访问权限
- [ ] **审计日志** - 所有工具调用完整记录

### P2.5: 统一错误码系统

**目标**：结构化错误处理，便于排查和前端展示

- [ ] **错误码定义** - 4 位数字错误码，分类明确
- [ ] **错误分类** - 系统错误、业务错误、配置错误、外部错误
- [ ] **错误信息** - 中英文双语错误消息
- [ ] **错误追踪** - 错误链、堆栈信息
- [ ] **HTTP 状态码映射** - 正确的 HTTP 状态码

**错误码规范**：
```
1xxx - 系统错误
2xxx - 配置错误
3xxx - 业务错误
4xxx - 外部服务错误
5xxx - 插件错误
```

---

### P2.8: 任务队列 / 长时任务

**目标**：支持长时间运行的异步任务

- [ ] **任务队列** - 异步任务调度、优先级队列
- [ ] **任务状态跟踪** - 排队中/执行中/已完成/失败
- [ ] **任务取消** - 支持中途取消长任务
- [ ] **进度回调** - 实时推送任务进度
- [ ] **失败重试** - 可配置的重试策略
- [ ] **任务持久化** - 服务重启后恢复任务

## 🌟 长期愿景 (v3.0+)

### 多 Agent 编排增强（v0.3.0 基础框架已完成，以下为未来演进方向）

> v0.3.0 已完成：三层架构、预定义链路、关键字匹配专家、责任链串行执行、规则引擎三层校验。
> 以下为未来增强方向，按优先级排列，后续版本逐步演进。

#### P0: 动态负载均衡

- [ ] **专家负载跟踪** - `ExpertPerformance` 增加 `current_load` 字段
- [ ] **执行前后负载增减** - `execute()` 入口 +1，出口 -1
- [ ] **负载分实际计算** - 当前 fallback 到默认值 1.0，需改为真实负载比

#### P1: 知识库检索引擎增强

- [ ] **向量检索实现** - 专家私有知识库的实际向量索引与检索
- [ ] **两级融合排序** - 先查私有库 → 再查公共库 → 结果融合排序（设计已有，代码未落地）
- [ ] **知识库持久化** - 私有知识标签 `Vec<String>` 改为实际可检索的向量存储

#### P1: 三级记忆模型完善

- [ ] **任务级共享记忆写入** - 策略执行时，将中间输出、评审意见、子任务结果写入共享记忆
- [ ] **记忆上下文传递** - 从当前 LLM system message 拼接，改为结构化的共享记忆读写
- [ ] **任务结束归档清理** - 任务完成后自动归档共享记忆，释放内存

#### P1: Pipeline 依赖图优化

- [ ] **子任务依赖拓扑排序** - 当前按领域列表顺序构建阶段，应改为基于 `SubTask.dependencies` 的拓扑排序
- [ ] **并行阶段支持** - 无依赖的阶段应可并行执行（当前全部串行）

#### P2: 专家历史表现持久化

- [ ] **数据库存储** - `ExpertPerformance` 从内存 HashMap 持久化到 PostgreSQL
- [ ] **跨会话恢复** - 服务重启后恢复专家历史数据

#### P2: 子任务解析增强

- [ ] **结构化输出** - LLM 子任务拆解从自由文本解析改为 JSON schema 约束
- [ ] **解析容错与重试** - 格式不对时自动重试（最多 2 次）

#### P2: 满意度判断鲁棒性

- [ ] **语义判断替代关键词匹配** - 当前硬编码关键词，改为 LLM 结构化判断
- [ ] **早停优化** - 连续两轮无明显改进时提前终止

### 配置中心

- [ ] **动态配置** - 配置热更新，无需重启
- [ ] **配置版本管理** - 配置变更历史、回滚
- [ ] **多环境支持** - 开发/测试/生产环境配置隔离
- [ ] **配置加密** - 敏感配置项加密存储

### 向量数据库抽象层

- [ ] **多后端支持** - pgvector / Milvus / Qdrant / Weaviate

- [ ] **混合检索** - 关键词 + 向量融合检索

### Agent 模板市场

- [ ] **模板仓库** - 官方 + 社区模板中心
