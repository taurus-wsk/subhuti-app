## 🎯 近期目标 (v1.1 - v1.5)

### P0: 示例专家插件（最高优先级）

**目标**：开发 2-3 个高质量示例专家插件，展示框架能力

- [ ] **心理咨询专家插件** - 完整展示人格注入 + 情感记忆 + CBT 技术
- [ ] **编程助手插件** - 展示代码理解 + 文件操作 + 多轮对话
- [ ] **学习伙伴插件** - 展示记忆管理 + 进度追踪 + 个性化推荐
- [ ] **插件开发模板** - 一键创建新插件的脚手架

**预期效果**：
- 验证插件系统的完整性
- 为其他开发者提供参考范例
- 展示框架的差异化能力

---

### P1: 数据库深度整合

**目标**：将心灵宫殿和记忆系统完全持久化到 PostgreSQL + pgvector

- [ ] **记忆持久化** - 所有记忆存储到 PostgreSQL
- [ ] **向量搜索** - 使用 bge-m3 生成向量，pgvector 进行语义搜索
- [ ] **人格持久化** - 性格数据、演化历史存数据库
- [ ] **多用户支持** - 按 user_id 隔离数据
- [ ] **迁移工具** - 内存数据迁移到数据库

**技术方案**：
```
内存模式（开发/测试）  ←→  数据库模式（生产）
     Memory                    DatabaseStore
       ↓                            ↓
  MemoryPalace                MemoryPalace
  （相同接口）                 （相同接口）
```

---

### P1.5: 配置系统增强

**目标**：灵活的多层配置体系

- [ ] **环境变量支持** - 所有关键配置可通过环境变量设置
- [ ] **配置文件** - TOML/YAML 配置文件支持
- [ ] **分层配置** - 默认值 → 配置文件 → 环境变量 → 运行时修改
- [ ] **配置热加载** - 部分配置可在运行时修改
- [ ] **配置验证** - 启动时验证配置正确性

**配置项清单**：
```
LLM 配置    - provider, api_key, base_url, model
数据库配置  - url, pool_size, timeout
记忆配置    - short_term_limit, archive_threshold
心灵配置    - evolve_interval, importance_threshold
HTTP 配置   - port, cors, rate_limit
```

---

### P2: 更多内置技能

**目标**：丰富默认技能库，开箱即用

- [ ] **文件操作增强** - 写入文件、删除文件、文件元信息
- [ ] **网络搜索** - 对接搜索引擎（Serper、Bing 等）
- [ ] **代码执行** - Docker 沙箱执行 Python/JS 代码
- [ ] **定时提醒** - 持久化提醒、到期推送
- [ ] **翻译技能** - 多语言翻译
- [ ] **总结技能** - 长文本摘要
- [ ] **思维导图** - 生成结构化知识图谱

---

### P2.5: 统一错误码系统

**目标**：结构化错误处理，便于排查和前端展示

- [ ] **错误码定义** - 4 位数字错误码，分类明确
- [ ] **错误分类** - 系统错误、业务错误、配置错误、外部错误
- [ ] **错误信息** - 中英文双语错误消息
- [ ] **错误追踪** - 错误链、堆栈信息
- [ ] **HTTP 状态码映射** - 正确的 HTTP 状态码

**错误码规范**：
```
1xxx - 系统错误
2xxx - 配置错误
3xxx - 业务错误
4xxx - 外部服务错误
5xxx - 插件错误
```
