📚 两种调度策略详解

1. SimpleDispatch - 单专家直连 默认

  • 适用场景：单专家匹配任务（得分最高的专家处理）
  • 执行流程：直接匹配最优专家 → 执行任务 → 返回结果
  • 特点：快速响应，适合简单咨询

2. Pipeline - 串行流水线

  • 适用场景：多专家匹配任务（≥2个专家得分超过阈值）
  • 执行流程：专家A → 专家B → 专家C（串行执行，后一专家基于前一专家结果）
  • 特点：适合需要多领域协作的复杂任务
  • 触发方式：自动触发（多专家匹配）或外部传入

策略选择逻辑

  • 默认策略：SimpleDispatch（单专家直连）
  • 自动升级：如果 ≥2 个专家匹配得分超过阈值，自动升级为 Pipeline
  • 手动指定：通过 API 参数 `strategy` 强制指定策略（如 `"pipeline"`）
