# 2026-06-28

## 多Agent协调编排架构改造完成

### 已完成工作
- 创建了完整的 `orchestrator` 模块 (`crates/subhuti/src/orchestrator/`)
  - `mod.rs`: 核心编排器，包含任务画像、专家匹配（加权多维度）、策略决策树
  - `strategies/simple_dispatch.rs`: 单专家直连策略
  - `strategies/pipeline.rs`: 串行流水线策略（阶段拆解+依赖传递）
  - `strategies/map_reduce.rs`: 并行发散-汇总策略（Map+Reduce）
  - `strategies/manager_worker.rs`: 主管-工人策略（动态拆解+汇总）
  - `strategies/critique_revise.rs`: 评审迭代策略（独立模式+叠加模式）

### 集成到 Subhuti 主结构
- `Subhuti` 新增 `orchestrator: tokio::sync::Mutex<Orchestrator>` 字段
- 新增方法：`run_orchestrated()`, `run_orchestrated_with_strategy()`, `analyze_task()`, `match_orchestrator_experts()`, `register_orchestrator_expert()` 等
- 使用 `tokio::sync::Mutex` 而非 `std::sync::Mutex`，支持在 async 上下文中持有锁

### HTTP API 新增端点
- `POST /subhuti/api/v1/orchestrate` - 编排器执行入口（支持自动/指定策略）
- `POST /subhuti/api/v1/orchestrate/analyze` - 任务画像分析
- `POST /subhuti/api/v1/orchestrate/match` - 专家匹配
- `GET /subhuti/api/v1/orchestrate/experts` - 编排器专家列表
