# Subhuti 框架改进路线图

## 一、当前框架短板分析

### 1. 缺少向量数据库集成
- **现状**：目前只有简单的关键词搜索，没有语义向量检索
- **对比**：LangChain/LlamaIndex 集成了 Pinecone、Chroma、FAISS 等向量数据库
- **影响**：无法实现真正的 RAG（Retrieval-Augmented Generation）

### 2. 缺少流式响应支持
- **现状**：目前只有同步/异步响应，没有流式输出
- **对比**：OpenAI API 原生支持流式，LangChain 有 Callback 机制
- **影响**：用户体验不佳，长响应需要等待

### 3. 缺少多模态支持
- **现状**：仅支持文本输入输出
- **对比**：GPT-4V、Gemini 支持图像输入
- **影响**：无法处理图片、语音等多模态内容

### 4. 缺少可视化工具
- **现状**：没有对话历史可视化、调试工具
- **对比**：LangChain UI、GPT-4 插件有可视化界面
- **影响**：调试困难，用户体验差

### 5. 缺少监控和可观测性
- **现状**：只有基本的日志，没有指标监控
- **对比**：LangChain 有 tracing、OpenTelemetry 集成
- **影响**：生产环境难以排查问题

### 6. 生态成熟度低
- **现状**：缺少社区贡献、文档、教程
- **对比**：LangChain 有庞大的社区和丰富的资源
- **影响**：学习成本高，遇到问题难以解决
