# Subhuti 框架正式设计规格（V1 轻量版）

## 定位

Rust 极简轻量 AI Agent 框架

## 设计哲学

- 薄封装（Harness 理念）
- 无魔法、无全局状态、可完全掌控
- 记忆分层、AI 自主检索记忆迷宫
- 类似 Gin：内核极简、能力插拔、中间件扩展

## 整体五层架构（固定）

```

├── lib.rs              # 主入口
```

## 1. 记忆层 Memory（核心亮点：记忆迷宫）

### 职责

所有数据存储、检索、归档、分层治理，与 LLM 运行完全解耦

### 三层标准记忆

| 层级 | 文件 | 功能 |
|------|------|------|
| 短期工作记忆 | `short_term.rs` | 当前对话上下文，自动注入 LLM |
| 长期归档记忆 | `long_term.rs` | 历史对话沉淀，AI 主动搜索 |
| 知识库语义记忆 | `knowledge.rs` | 向量知识、外部文档 |

### 对外标准能力

- 写入短期记忆
- 自动归档长期记忆
- 三层独立搜索函数（给 AI 当工具调用）
- 记忆裁剪、过期、摘要

## 2. 运行层 Runtime（框架内核，轻量底座）

### 职责

真正执行层，所有可运行能力、模型、工具、约束

### 包含模块

#### 2.1 LLM 抽象层 (`runtime/llm/`)

```rust
pub trait LLM: Send + Sync {
    fn provider(&self) -> LLMProvider;
    fn config(&self) -> &LLMConfig;
    async fn chat(&self, messages: Vec<Message>) -> Result<String>;
    async fn chat_streaming(&self, messages: Vec<Message>, callback: Box<dyn Fn(String) + Send>) -> Result<()>;
    async fn health_check(&self) -> Result<bool>;
}
```

**已实现客户端：**
- `OpenAIClient` - OpenAI API
- `OllamaClient` - 本地 Ollama
- `DoubaoClient` - 字节跳动豆包

#### 2.2 工具系统 (`runtime/tools/`)

```rust
pub trait Tool: Send + Sync {
    fn info(&self) -> ToolInfo;
    async fn run(&self, params: Value) -> Result<ToolResult>;
}
```

- 极简 Tool Trait：name / desc / schema / run
- 所有记忆搜索、外部能力全部是工具
- 内核零业务逻辑（Gin 思想）

#### 2.3 约束护栏 (`runtime/constraints.rs`)

- 最大工具调用轮次
- 最大上下文长度
- 超时控制
- 非法参数拦截

#### 2.4 会话 Session (`runtime/session.rs`)

- 所有状态归 Session
- Runtime 无状态（可水平扩展）

## 3. 流程层 Flow（Agent 智能闭环）

### Flow Trait - 流程抽象接口

```rust
#[async_trait]
pub trait Flow: Send + Sync {
    fn name(&self) -> &str;
    fn description(&self) -> &str;
    async fn execute(&self, ctx: &mut FlowContext<'_>) -> Result<String>;
}
```

### 内置流程

| 流程 | 文件 | 适用场景 |
|------|------|----------|
| `SimpleFlow` | `simple.rs` | 简单对话，无工具调用 |
| `ReactFlow` | `react.rs` | ReAct 循环，自动工具调用 |
| `PlanActFlow` | `plan_act.rs` | 先规划再执行 |

### FlowManager - 流程管理器

```rust
let mut flow_manager = FlowManager::new();

// 选择内置流程
flow_manager.set_flow_type(FlowType::Simple);  // 或 React, PlanAct

// 注册自定义流程
flow_manager.register_custom_flow(Arc::new(MyCustomFlow));
```

### 核心流程

```
Plan → Act → Observe → Reflect

Plan:   LLM 判断是否需要工具 / 记忆检索
Act:    执行工具、搜索记忆
Observe: 清洗、标准化工具返回结果
Reflect: 校验是否有效、是否需要重试检索
```

## 5. Skill 层（纯代码风格 + 预设主流程模板）

### 设计理念

**Skill = 纯代码实现 + 可选预设主流程模板**

```
用户输入 → Skill 路由匹配 → 执行 Skill.execute()（纯代码） → 返回
                ↓
         可选择预设 Flow 模板（ReAct、Plan-Act 等）
                ↓
         或完全自定义流程
```

### 核心创新

| 传统设计（有问题） | Subhuti 设计（新架构） |
|-------------------|----------------------|
| Skill 定义声明式步骤 | Skill 纯代码实现 |
| 表达能力受限 | 灵活性最大化 |
| 需要学习 FlowStep 语法 | 直接用 Rust 代码 |
| 动态逻辑困难 | 完全自定义控制流 |

### FlowTemplate - 预设主流程模板

```rust
pub enum FlowTemplate {
    /// ReAct 流程：Reasoning + Acting
    /// 适合需要多轮思考和工具调用的场景
    ReAct,

    /// Plan-Act 流程：Plan → Act → Observe
    /// 适合需要规划的任务
    PlanAct,

    /// Simple 流程：简单对话
    /// 适合简单的问答场景
    Simple,

    /// Chain-of-Thought 流程：思维链
    /// 适合需要复杂推理的场景
    ChainOfThought,
}
```

### Skill Trait 设计（纯代码风格）

```rust
#[async_trait]
pub trait Skill: Send + Sync + 'static {
    fn name(&self) -> &str;
    fn description(&self) -> &str { "" }
    fn matches(&self, input: &str) -> f32;

    /// 选择预设流程模板（可选）
    /// 返回 Some(template) 表示使用预设模板
    /// 返回 None 表示完全自定义
    fn flow_template(&self) -> Option<FlowTemplate> {
        None
    }

    /// 核心执行方法（纯代码实现）
    /// 这是 Skill 的核心功能：用代码实现业务逻辑
    async fn execute(&self, ctx: SkillContext<'_>) -> Result<String>;

    fn priority(&self) -> i32 { 0 }
}
```

### SkillContext - Skill 执行上下文

```rust
pub struct SkillContext<'a> {
    pub input: &'a str,
    pub session: &'a mut Session,
    pub runtime: &'a Runtime,
    pub memory: &'a Memory,
    pub confidence: f32,
    pub flow_template: Option<FlowTemplate>,
}

impl<'a> SkillContext<'a> {
    /// 调用工具
    pub async fn call_tool(&self, name: &str, args: Value) -> Result<String>;

    /// 调用 LLM
    pub async fn call_llm(&self, messages: Vec<Message>) -> Result<String>;

    /// 获取记忆
    pub fn get_memory(&self, key: &str) -> Option<String>;

    /// 设置记忆
    pub fn set_memory(&self, key: &str, value: &str);
}
```

### Skill 执行流程

```
Skill 匹配成功:
    ↓
创建 SkillContext
    ↓
调用 skill.execute(ctx)
    ↓
    ├── 使用预设模板？ → 按模板流程执行
    │   ├── ReAct → 多轮思考 + 工具调用
    │   ├── PlanAct → 规划 + 执行 + 观察
    │   ├── Simple → 简单对话
    │   └── ChainOfThought → 思维链推理
    │
    └── 完全自定义？ → 纯代码实现
    │   ├── 调用 ctx.call_tool()
    │   ├── 调用 ctx.call_llm()
    │   ├── 调用 ctx.memory 操作
    │   ├── 自定义业务逻辑
    │   └── 条件判断、循环等
    ↓
返回结果

Skill 匹配失败:
    ↓
使用默认 Flow（AI 自主决策）
```

### 使用示例

#### 示例1：使用预设 ReAct 流程模板

```rust
use subhuti::skill::{Skill, SkillContext, FlowTemplate};

struct WeatherSkill;

#[async_trait]
impl Skill for WeatherSkill {
    fn name(&self) -> &str { "weather" }

    fn matches(&self, input: &str) -> f32 {
        if input.contains("天气") { 0.9 } else { 0.0 }
    }

    // 选择预设流程模板
    fn flow_template(&self) -> Option<FlowTemplate> {
        Some(FlowTemplate::ReAct)
    }

    // 纯代码实现
    async fn execute(&self, ctx: SkillContext<'_>) -> Result<String> {
        // 步骤1：调用工具查询天气
        let weather_data = ctx.call_tool("get_weather", json!({"city": "北京"})).await?;

        // 步骤2：调用 LLM 生成回复
        let response = ctx.call_llm(vec![
            Message::user(format!("根据天气数据 {} 回答用户问题", weather_data))
        ]).await?;

        Ok(response)
    }
}
```

#### 示例2：完全自定义流程

```rust
struct OrderSkill;

#[async_trait]
impl Skill for OrderSkill {
    fn name(&self) -> &str { "order" }

    fn matches(&self, input: &str) -> f32 {
        if input.contains("下单") { 0.9 } else { 0.0 }
    }

    // 不使用预设模板，完全自定义
    fn flow_template(&self) -> Option<FlowTemplate> {
        None // None 表示完全自定义流程
    }

    async fn execute(&self, ctx: SkillContext<'_>) -> Result<String> {
        // 步骤1：解析订单信息（使用 LLM）
        let order_info = ctx.call_llm(vec![
            Message::user(format!("从以下文本中提取订单信息：\n{}", ctx.input))
        ]).await?;

        // 步骤2：检查库存（调用工具）
        let inventory_result = ctx.call_tool(
            "check_inventory",
            json!({"order": order_info})
        ).await?;

        // 步骤3：业务逻辑判断
        if inventory_result.contains("库存不足") {
            return Ok(format!("抱歉，{}", inventory_result));
        }

        // 步骤4：创建订单（调用工具）
        let order_result = ctx.call_tool(
            "create_order",
            json!({"order": order_info})
        ).await?;

        // 步骤5：记录到记忆
        if order_result.contains("成功") {
            ctx.set_memory("last_order", &order_result);
        }

        Ok(format!("订单处理结果：{}", order_result))
    }
}
```

### 优势对比

| 特性 | 声明式 Skill（旧设计） | 纯代码 Skill（新设计） |
|------|---------------------|---------------------|
| 执行方式 | 声明式 FlowStep | 纯代码实现 |
| 灵活性 | 中（受 FlowStep 限制） | 高（完全自定义） |
| 学习曲线 | 需学习 FlowStep 语法 | 直接用 Rust 代码 |
| 动态逻辑 | 困难（条件、循环受限） | 容易（代码级控制） |
| 复杂业务 | 表达能力受限 | 完全支持 |
| AI 消耗 | 可控（预设步骤） | 可控（代码控制） |
| 精确度 | 中 | 高（代码级精确） |

### 混合架构优势

**两种方式共存，按需选择：**

- **简单场景**：使用预设 Flow 模板（ReAct、Simple 等）
  - 快速开发
  - 标准化流程
  - 减少 AI 消耗

- **复杂场景**：完全自定义流程
  - 精确控制
  - 复杂业务逻辑
  - 完全灵活性

### 6.2 请求处理生命周期（核心）

每次用户请求的完整生命周期：

```
用户输入
    ↓
┌───────────────────────────────────────┐
│ [Extension] before_prompt            │
│   - 上下文预处理、裁剪               │
└───────────────────────────────────────┘
    ↓
┌───────────────────────────────────────┐
│ [Memory] 写入短期记忆                 │
│   - session.add_message(User)        │
└───────────────────────────────────────┘
    ↓
┌───────────────────────────────────────┐
│ [Skill] 路由匹配                     │
│   - 遍历所有 Skill                   │
│   - 计算匹配度 (0.0-1.0)             │
│   - 获取预设 flow_steps              │
└───────────────────────────────────────┘
    ↓
┌───────────────────────────────────────────────────────────────┐
│  Skill 匹配成功，有预设步骤？                                  │
└───────────────────────────────────────────────────────────────┘
    ↓ Yes                          ↓ No
┌────────────────────┐      ┌─────────────────────────────┐
│ Flow.execute_steps │      │ Flow.execute()              │
│ (预设步骤执行)      │      │ (AI 自主 ReAct 循环)        │
│                    │      │                             │
│ - Tool             │      │ - Plan → Act                │
│ - Knowledge        │      │ - Observe → Reflect         │
│ - LLM              │      │ - 自动工具调用               │
│ - Condition        │      │                             │
└────────────────────┘      └─────────────────────────────┘
    ↓                          ↓
┌───────────────────────────────────────┐
│ [Memory] 写入响应                     │
│   - session.add_message(Assistant)    │
└───────────────────────────────────────┘
    ↓
┌───────────────────────────────────────┐
│ [Extension] after_complete           │
│   - 自动归档记忆                      │
│   - 统计、日志                        │
└───────────────────────────────────────┘
    ↓
返回响应
```

### 6.3 Extension Hooks 生命周期

| Hook | 触发时机 | 用途 |
|------|----------|------|
| `before_prompt` | 请求开始前 | 上下文预处理、裁剪、敏感词过滤 |
| `before_tool` | 工具调用前 | 工具校验、权限检查、日志 |
| `after_tool` | 工具调用后 | 结果摘要、缓存、异常处理 |
| `after_complete` | 请求结束后 | 自动归档记忆、统计、日志 |

### 6.4 Session 生命周期

```
Session 创建 → 多次请求 → Session 销毁
     ↓              ↓           ↓
  new()         run()      drop()
     ↓              ↓           ↓
  初始化         读写记忆      资源释放
  状态重置       执行逻辑      归档长期记忆
```

### 6.5 核心代码实现

```rust
pub async fn run(&self, session: &mut Session, input: &str) -> Result<String> {
    // 1. Extension: before_prompt
    self.extensions.call_before_prompt(session, input).await?;

    // 2. Memory: 写入短期记忆
    session.add_message(Role::User, input);

    // 3. Skill 路由匹配
    let response = if let Some((skill_match, steps)) = self.skills.get_matched_skill(input) {
        if steps.is_empty() {
            // Skill 匹配成功但无预设步骤，使用默认 Flow
            tracing::info!("Skill {} matched, no steps, using default Flow", skill_match.info.name);
            self.flow.execute(session, &self.runtime, &self.memory).await?
        } else {
            // 使用预设步骤执行
            tracing::info!("Skill {} matched, executing preset steps", skill_match.info.name);
            self.flow.execute_steps(session, &self.runtime, &self.memory, input, &steps).await?
        }
    } else {
        // 没有 Skill 匹配，使用默认 Flow（AI 自主决策）
        tracing::info!("No skill matched, using default Flow (AI autonomous)");
        self.flow.execute(session, &self.runtime, &self.memory).await?
    };

    // 4. Memory: 写入响应
    session.add_message(Role::Assistant, &response);

    // 5. Extension: after_complete
    self.extensions.call_after_complete(session).await?;

    Ok(response)
}
```

### FlowContext 上下文存储

```rust
pub struct FlowContext<'a> {
    pub session: &'a mut Session,
    pub runtime: &'a Runtime,
    pub memory: &'a Memory,
    pub config: FlowConfig,
    pub state: FlowState,
    pub iteration: usize,
    pub input: &'a str,
    /// 上下文存储（用于保存步骤执行结果）
    pub context_data: HashMap<String, String>,
}

impl FlowContext {
    /// 保存上下文数据
    pub fn save_context(&mut self, key: &str, value: &str) {
        self.context_data.insert(key.to_string(), value.to_string());
    }

    /// 获取上下文数据
    pub fn get_context(&self, key: &str) -> Option<&String> {
        self.context_data.get(key)
    }
}
```

### 6.6 设计特点

| 特点 | 说明 |
|------|------|
| 可观测性 | 每个阶段都可以通过 Extension Hook 监控 |
| 可扩展性 | 生命周期的每个节点都可以插入自定义逻辑 |
| 无状态设计 | Runtime 无状态，Session 保存所有状态 |
| 优雅降级 | Skill 匹配失败自动 Fallback 到 Flow |
